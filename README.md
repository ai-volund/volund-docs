# VOLUND Documentation

VOLUND is a multi-tenant, general-purpose AI agent platform running on Kubernetes.

## Repositories

| Repo | Description |
|------|-------------|
| [volund](https://github.com/ai-volund/volund) | Platform core — API gateway + control plane + Forge API |
| [volund-operator](https://github.com/ai-volund/volund-operator) | Kubernetes operator — AgentInstance, AgentWorkspace, TenantNamespace CRDs |
| [volund-agent](https://github.com/ai-volund/volund-agent) | Agent runtime — orchestrator and specialist agent execution |
| [volund-proto](https://github.com/ai-volund/volund-proto) | Shared protobuf definitions, CloudEvent types, schemas |
| [volund-forge](https://github.com/ai-volund/volund-forge) | Forge CLI and SDK — skill and agent profile development |
| [volund-desktop](https://github.com/ai-volund/volund-desktop) | Desktop app — cross-platform Tauri 2.x client with shared React frontend |
| [volund-docs](https://github.com/ai-volund/volund-docs) | Documentation, architecture decisions, and guides |

## Diagrams

All architecture diagrams are authored in Excalidraw. Each diagram has:
- An **inline SVG** rendered in the markdown docs (auto-generated by CI)
- An **interactive version** hosted on excalidraw.com (linked from each doc)
- A **source JSON file** in `architecture/diagrams/` (editable in any Excalidraw client)

A [GitHub Action](.github/workflows/export-diagrams.yml) automatically converts `.excalidraw.json` files to SVG on push using [excalirender](https://github.com/JonRC/excalirender).
